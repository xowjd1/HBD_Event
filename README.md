# 🎂 HBD Event

> 생일·기념일에 사용하는 **심리 테스트 기반 인터랙티브 이벤트 앱**  
> 유저 입력 → 질문 응답 → 결과 화면 → CSV 로그 저장까지 한 번에 처리하는 **로컬 이벤트용 템플릿**

---

## 🎮 프로젝트 개요

- **프로젝트 명** : HBD Event  
- **프로젝트 타입** : 생일/기념일 심리 테스트 & 이벤트용 인터랙티브 콘텐츠  
- **개발 형태** : 개인 프로젝트  
- **개발 기간** : 2025년 10월 24일 ~ 2025년 11월 01일
- **개발 인원** : 개발 1명
- **개발 도구** : Unity 6, Github Desktop, Rider

생일 파티, 매장 이벤트, 오프라인 부스 등에서  
간단한 **심리 테스트 / 설문**을 진행하고, 그 자리에서 바로 결과를 보여주며  
동시에 **유저 데이터를 CSV로 쌓아 분석에 활용**할 수 있도록 제작한 Unity 프로젝트입니다.

닉네임, 생년월일을 입력하고 몇 가지 질문에 답하면,  
유저는 본인의 타입/메시지가 담긴 결과 화면을 확인하고,  
운영자는 백그라운드에서 자동으로 쌓인 **유저의 데이터**를 얻을 수 있습니다.

## 🧪 콘텐츠 구성

### 1. 시작 폼 (Start Form)

- 닉네임 입력 (텍스트 필드)
- 생년월일 입력 (텍스트 필드)
- 모든 필수 값이 올바르게 입력되었을 때만 **다음 버튼 활성화**
  - 빈 값 / 미선택 방지 유효성 검사
  - 닉네임은 최소 1자, 생년월일은 실제 날짜만 가능

### 2. 질문 & 응답 흐름

- 객관식 버튼 기반 질문 UI
- 각 질문은 **버튼 선택 → 내부 점수/타입 정보 누적** 방식으로 처리

### 3. 결과 화면

- 선택 정보를 결과 화면에서 출력
- 자동 스크린 샷 기능으로 빌드 폴더에 'ScreenShot' 폴더를 생성하고 내부에 스크린샷을 저장

### 4. 데이터 저장 (CSV 로그)

- 유저가 결과 화면까지 도달했을 때,  
  `UserData` 폴더에 **CSV 포맷으로 응답 기록 저장**

## 👨‍💻 담당 업무

### 🧩 게임/앱 로직 구현

- `StartForm` 검증 로직 구현
  - 닉네임, 생년월일 필수 입력 체크
  - 입력이 유효할 때만 다음 단계로 진행
- 질문/응답 로직
  - 각 질문마다 버튼 클릭 시 내부 점수 또는 타입 인덱스 갱신
  - 질문 진행 상태(현재 인덱스) 관리 및 마지막 질문 도달 시 결과 화면으로 전환
- 스트레스 결과 타입 계산
  - 누적 값에 따라 스트레스 결과 타입 결정
  - 결과 타입별 텍스트/이미지를 바인딩하는 구조 설계

### 📊 데이터 로깅 & 파일 I/O

- 유저 응답을 **CSV 포맷**으로 직렬화해 `UserData` 폴더에 저장
- 추후 엑셀/구글 스프레드시트로 열어
  - 참여자 수 집계
  - 연령대별 응답 경향 분석
  - 결과 타입 분포 시각화
  등이 가능하도록 컬럼 구성

### 🛠 에디터 & 운영 편의 기능

- **스크린샷 캡처 기능** 구현
 - 마지막 페이지 진입 시 자동으로 현재 게임 화면을 캡처하여 `ScreenShot` 폴더에 PNG 파일로 저장
